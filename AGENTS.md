# Правила работы над проектом

## Описание проекта и целей пользователя

- Проект: ContentZavod Template — конвейер по производству обучающих материалов об ИИ-инструментах.
- Цель: быстро развернуть проект, где AI-агент помогает пройти путь от идеи мастер-класса до видео, статьи и проверки качества.
- Как используется: пользователь вызывает skill `/mk`, агент создаёт артефакты в `content/<slug>/`, человек записывает экран и принимает решения в контрольных точках.
- Подробнее: `docs/concept.md`, `docs/guides/content-pipeline.md`.

## Назначение AGENTS.md и границы

- AGENTS.md содержит только универсальные правила.
- Специфические инструкции хранить в `docs/` или уточнять у пользователя; не раздувать AGENTS.md.
- Все проектные документы хранить в `docs/`, кроме `README.md`, `AGENTS.md` и `CLAUDE.md`.

## При начале работы над задачей

- Прочитать блок "Описание проекта и целей пользователя".
- Прочитать последние записи `docs/development-history.md`, если задача продолжает предыдущую работу.
- По карте документов `docs/index.md` открыть только релевантные файлы.
- Если есть локальный `MEMORY.md` с незавершённой работой, начать с него. В шаблоне этот файл не обязателен.
- При нехватке контекста задать один короткий вопрос.

## Универсальные правила

- Следовать существующим паттернам, правилам и соглашениям проекта.
- Не использовать AGENTS.md как стайлгайд или список команд; для деталей есть `docs/guides/`.
- Документы держать краткими; вместо копирования кода ссылаться на файлы и модули.
- В публичных/коммитимых файлах не хардкодить абсолютные пути, реальные ключи, личные имена, приватные URL и данные клиентов.
- Если планируются большие изменения — сначала кратко согласовать подход.
- Обновлять затронутую документацию.

## Контроль размера документов

- Лимит для обычных `.md` в `docs/`: 200 строк.
- Для гайдов при приближении к лимиту сжимать текст или выносить крупную секцию в отдельный файл.
- Для накопителей (`development-history.md`, `audit-issues.md`) старые записи архивировать или сжимать.

## Чек-лист перед завершением задачи

- Проверить, что выполнены требования задачи.
- Запустить релевантные тесты или явно зафиксировать, почему они не запускались.
- Обновить документацию и `docs/development-history.md`.
- Если принято важное решение — добавить запись в `docs/adr.md`.
- Свериться с `docs/guides/dod.md`.
