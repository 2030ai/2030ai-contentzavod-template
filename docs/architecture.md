# Архитектура: пайплайн производства контента

Публичный шаблон строит два результата из одной идеи: видеоурок и статью со скриншотами.

## Что делает агент

- Генерирует или принимает идею мастер-класса.
- Валидирует идею через синтетическую фокус-группу из 8 персон.
- Пишет бриф, сценарий и ТЗ на запись экрана.
- Анализирует записанное видео по кадрам.
- Готовит `voiceover-script.md` и `edit.py`.
- Собирает видео через `pipeline/pipeline/editor.py`.
- Проверяет монтаж и пишет статью.

## Что делает человек

- Принимает решения в паузах пайплайна: выбрать идею, утвердить сценарий, принять итог.
- Записывает экран по `recording-brief.md`.
- Проверяет, что в записи нет приватных данных.
- Публикует готовый контент на своей площадке, если CMS-интеграция не добавлена.

## Маршруты

**Фаза 1 — валидация идеи:**

Тема или идея -> исследователь -> фокус-группа -> контролёр -> бриф -> пауза.

**Фаза 2 — производство:**

Сценарий -> ТЗ записи -> запись человеком -> анализ видео -> монтаж + TTS -> контроль монтажа -> статья -> финальная проверка -> очистка.

Полное описание: `docs/guides/content-pipeline.md`.

## Ограничения

- Запись экрана не автоматизирована: человек записывает видео вручную.
- Для качественного анализа держите raw-видео до 10 минут.
- `ffmpeg` и `ffprobe` должны быть установлены локально.
- Интеграция с CMS/YouTube не входит в шаблон.
