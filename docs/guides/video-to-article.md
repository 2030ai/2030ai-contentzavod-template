# Гайд: видео → страница (article.md)

Как из готового видеоурока сделать страницу с пошаговой инструкцией, скриншотами и копируемыми промптами.

## Вход

- Финальное видео (уже с кропом, 1920x1080)
- `voiceover-script.md` — сценарий озвучки
- `edit.py` — массив `SEGMENTS` с таймкодами, скоростями и текстом озвучки

## Процесс

### 1. Рассчитать таймкоды в финальном видео

Сегменты в edit.py содержат исходные таймкоды из записи. В финальном видео длительность каждого сегмента определяется TTS-аудио (видео растягивается/сжимается под озвучку).

```bash
# Получить реальные длительности сегментов
cd segments/
for f in *_combined.mp4; do
  dur=$(ffprobe -v quiet -print_format json -show_format "$f" \
    | python3 -c "import sys, json; print(f'{float(json.loads(sys.stdin.read())[\"format\"][\"duration\"]):.2f}')")
  echo "$f: ${dur}s"
done
```

Суммировать длительности нарастающим итогом → получить начало и конец каждого сегмента в финальном видео.

### 2. Извлечь скриншоты

По 1 ключевому кадру на шаг. Таймкод = середина сегмента (или ближе к концу, если результат виден в конце).

```bash
ffmpeg -y -ss TIMECODE -i video.mp4 -frames:v 1 -update 1 step-NN.png
```

Сохранять в `article-images/step-01.png`, `step-02.png`, и т.д.

Какие сегменты пропускать:
- Вступление и заключение — нет визуально полезного контента
- Сегменты генерации (ускоренные) — промежуточное состояние, неинформативно

Какие сегменты брать:
- Ввод данных (запросы, инструкции) — пользователь видит что набирать
- Готовые результаты — пользователь видит что получится

### 3. Извлечь промпты

Из `voiceover-script.md` и `SEGMENTS` найти все тексты, которые пользователь вводил в интерфейс. Это:
- Инструкции проекта/системные промпты
- Запросы к ИИ

Оформить как копируемые блоки в markdown: **заголовок** + код-блок на верхнем уровне.
НЕ вкладывать код-блоки в blockquote (`> ``` ... ````) — это ломает HTML-рендереры.
Правильный формат: `**Промпт:**` + пустая строка + ` ``` текст ``` `.

### 4. Написать текст

Переписать озвучку из повествовательного стиля в формат инструкции:
- «Мы делаем...» → «Сделайте...»
- «Мы нажимаем...» → «Нажмите...»
- «Смотрим результат» → «Посмотрите результат»

Объединить смежные сегменты в шаги по смыслу. Например, «набор запроса» + «генерация» + «готовый результат» = один шаг с двумя подразделами.

### 5. Собрать article.md

Структура:

```markdown
# Заголовок статьи

<!-- video: filename.mp4 -->

---

## Пошаговая инструкция

### Шаг 1. Название шага
Текст инструкции...

![Описание](article-images/step-01.png)

**Промпт:**

```
Текст промпта
```

### Шаг 2. Название шага
...

## Итого
Краткое резюме: что получили, зачем это нужно.
```

## Выход

- `article.md` — готовая статья
- `article-images/step-NN.png` — скриншоты из видео (1920x1080)

## Чек-лист

- [ ] Скриншоты 1920x1080, ключевой момент каждого шага
- [ ] Все промпты из видео оформлены как копируемые блоки
- [ ] Текст в формате инструкции (императив), не повествование
- [ ] Структура: H1 → видео → шаги (H3 + текст + скриншот + промпт) → итого
- [ ] Статья самодостаточна — можно следовать без видео
