# Гайд: написание текста озвучки

Правила для написания `voice:` текстов в сценариях и `voiceover-script.md`.

## Видео первично, текст вторичен

Озвучка пишется ПОСЛЕ просмотра записанного видео (шаг 7 — Монтажёр).
AI извлекает кадры, смотрит что происходит на экране и пишет текст под это.

**Принцип:** описывай то, что ВИДНО. Если на экране таблица с данными —
назови конкретные цифры. Если виден результат генерации — прокомментируй
ключевые пункты. `scenario.md` — ориентир, но реальное видео важнее.

**Speed factor:** для каждого сегмента проверяй отношение
`video_duration_after_speed / audio_duration`. Цель: 0.7–1.5x.
Если мало видео и много текста — расширь границы сегмента.
Если много видео и мало текста — сузь границы.
Подробнее: `pipeline-phase2-production.md`, шаг 7.

## Главное правило: пиши как говоришь

Текст озвучки — это устная речь. Он должен звучать естественно при чтении вслух. Если фраза выглядит нормально на письме, но звучит странно — переписать.

## Аббревиатуры и английские термины

**Не использовать аббревиатуры в озвучке.** Заменять полными русскими фразами.

TTS-движки могут неправильно произносить аббревиатуры в русском тексте. ElevenLabs справляется лучше, но не идеально. Фонетический словарь (`text_prep.py` + `ENGINE_OVERRIDES`) — страховка, а не решение.

| Не писать | Писать |
|-----------|--------|
| SEO | поисковое продвижение |
| SEO-описание | описание для поисковиков |
| H1, H2, H3 | заголовок первого/второго/третьего уровня |
| meta description | описание страницы для поисковиков |
| API | программный интерфейс |
| URL | ссылка / адрес страницы |
| PDF | документ / файл |
| FAQ | частые вопросы |
| CTA | призыв к действию |
| B2B | для бизнеса |
| UX | удобство использования |
| KPI | показатели эффективности |

**Названия продуктов:** писать строчными русскими буквами. ElevenLabs чувствителен к регистру — заглавные буквы могут сдвинуть ударение (например, «Курсор» → ударение на У, а «курсор» → правильно на О). Проблемные слова фиксируются в `ENGINE_OVERRIDES` в `text_prep.py`.

| Не писать | Писать |
|-----------|--------|
| Cursor | курсор |
| Claude Code | клауд код |
| Claude | клауд |
| ChatGPT | чат джипити |
| Gemini | джемини |
| DeepSeek | дипсик |
| YouTube | Ютуб |
| Google | Гугл |
| Perplexity | перплексити |

Проверять на слух перед финальной сборкой.

**На экране видно оригинал.** Зритель видит «SEO» в интерфейсе, а голос объясняет «поисковое продвижение» — это даже лучше для обучения.

## Стиль

- Короткие предложения (1–2 строки). Длинные — разбивать.
- Повествовательный тон: «мы делаем», «смотрим», «нажимаем». Не инструктивный (это для статьи).
- Не начинать предложение с «Итак», «Далее», «Затем» подряд — однообразно.
- Паузы: точка = пауза. Если нужна пауза внутри предложения — тире `—`.

## Запрещённые символы

**Не использовать `--` (двойной дефис) в тексте озвучки.** TTS-движки могут прочитать его буквально или создать артефакты. Вместо `--` использовать:
- Точку `.` — для паузы между предложениями
- Тире `—` — для паузы внутри предложения
- Запятую `,` — для короткой паузы

| Плохо | Хорошо |
|-------|--------|
| бесплатный и платный -- и покажу | бесплатный и платный, и покажу |
| В бесплатном -- одна модель | В бесплатном — одна модель |
| дальше всё изменится -- но об этом позже | дальше всё изменится. Но об этом позже |

## Правила произношения ElevenLabs

ElevenLabs чувствителен к регистру — заглавные буквы могут сдвинуть ударение. Проверенные правила:

| Слово | Проблема | Как пишем в тексте | Как фиксим в коде |
|-------|----------|-------------------|-------------------|
| Курсор | «КУрсор» вместо «курсОр» | курсор (строчная) | ENGINE_OVERRIDES → курсор |
| Джипити | «джипИти» вместо «джипитИ» | джипити (строчная) | ENGINE_OVERRIDES → ДжиПиТИ |
| ChatGPT | то же | Чат Джи-Пи-Ти (через дефисы) | PHONETIC_DICT |
| дипсик | ударение не на И | дипсик | ENGINE_OVERRIDES → дипсИИк |
| джемини | произносит по-русски | джемини | ENGINE_OVERRIDES → Джеминай |
| клауд | «клауд» вместо «клод» | клауд | ENGINE_OVERRIDES → клод |
| SEO | «сео» вместо «сэо» | лучше «поисковое продвижение» | ENGINE_OVERRIDES → СьЭо |
| URL | «ю-ар-эл» | лучше «ссылка» | PHONETIC_DICT → УРЛ |

**При обнаружении нового проблемного слова:**
1. Добавить в `ENGINE_OVERRIDES` в `text_prep.py` (автоматическая замена).
2. Добавить в `ELEVENLABS_WATCHLIST` там же (валидатор предупредит при сборке).
3. Добавить в таблицу выше (документация для человека).

## Как избежать галлюцинаций ElevenLabs

ElevenLabs Multilingual v2 иногда вставляет мусорные слоги — особенно вокруг тире и коротких слов. Пайплайн автоматически проверяет каждый TTS-файл через Whisper и перегенерирует при ошибке (до 3 раз). Если после 3 попыток мусор остаётся — нужно перефразировать текст.

**Правила, снижающие риск галлюцинаций:**

| Рискованно | Безопаснее |
|------------|------------|
| Ключ — это пропуск | Ключ это ваш пропуск |
| Модель — нано банана | Модель нано банана |
| Готово — результат | Готово. Вот результат |

- Избегать тире `—` между короткими словами (1–2 слога). Заменять точкой или убирать.
- Разбивать длинные предложения (>20 слов) на два.
- Транслитерации («опенроутер», «нано банана») — безопасны сами по себе, но в сочетании с тире рядом могут провоцировать глюк.

## Фонетический словарь (text_prep.py)

Три слоя защиты произношения:
1. **PHONETIC_DICT** — общие замены для всех движков (английский → русская фонетика).
2. **ENGINE_OVERRIDES** — замены, разные для Yandex и ElevenLabs.
3. **ELEVENLABS_WATCHLIST** + `validate_tts_text()` — валидатор, который при сборке выводит предупреждения о словах с известными проблемами.

Валидатор встроен в `editor.py` — запускается автоматически перед каждой генерацией TTS.
