# Критерии завершенности (DoD)

Минимальный DoD для любой задачи:

- [ ] Функционал работает согласно требованиям
- [ ] Тесты проходят; добавлены новые при необходимости (или указано почему нет)
- [ ] Документация обновлена, включая `docs/development-history.md`

Дополнительно по типам задач (если применимо):

Новая фича:
- [ ] Обновлен CHANGELOG (если ведется)
- [ ] Обновлена документация пользователя

Багфикс:
- [ ] Выявлена root cause
- [ ] Добавлен регрессионный тест

Рефакторинг:
- [ ] Функциональность не изменена (тесты подтверждают)
- [ ] ADR создан при значимых изменениях

Документация:
- [ ] Информация актуальна
- [ ] Примеры кода добавлены
- [ ] Ссылки работают
- [ ] Затронутые `.md` файлы в `docs/` ≤ 200 строк (см. AGENTS.md «Контроль размера документов»)

Создание мастер-класса (контент):
- [ ] Идея прошла фокус-группу (вердикт «В работу»)
- [ ] Сценарий соответствует `scenario-structure.md` (вступление, шаги, лайфхак, затравка, заключение)
- [ ] Тайминг 4:30–5:30, шагов 5–12
- [ ] Артефакты в `content/<slug>/` (brief, scenario, recording-brief — минимум)
- [ ] Видео записано и прошло контроль (шаг 6)
- [ ] Монтаж проверен покадрово (шаг 8 — Контролёр)
- [ ] Статья написана (шаг 9 — Редактор)
- [ ] `content/README.md` обновлён
- [ ] `pipeline/content-registry.md` обновлён
