## Шаг 4. Редактор — сценарий

**Роль:** автор обучающих видеосценариев.
**Вход:** `brief.md` из шага 3 (одобренная пользователем идея).

### Действия
1. Прочитать `docs/guides/scenario-structure.md` и `docs/guides/voiceover-writing.md`.
2. Написать сценарий по структуре:
   - Вступление (10-15 сек): проблема + что получим
   - Основные шаги (3:00-3:30): 5-12 шагов, каждый = действие -> генерация -> результат
   - Лайфхак (15-20 сек): неочевидная возможность или частая ошибка
   - Заключение (10-15 сек): что получили + зачем это нужно
   - Затравка «Следующий уровень» (15-20 сек): расширение через Cursor + Claude Code + MCP
3. Для каждого шага: действие, генерация, результат (мин. 8 сек), текст озвучки, промпт/текст для ввода.
   Формат промптов в scenario.md — как код-блок для копирования:
   ```
   ### Шаг N. Название
   Действие: ...
   **Промпт:** `текст промпта для ввода`
   Озвучка: ...
   ```
4. Написать ЧЕРНОВИК текста озвучки (ориентир для записи и монтажа).
   Финальная озвучка пишется на шаге 7 после просмотра реального видео.

### Самопроверка
- [ ] Структура и тайминг 4:30-5:30? Без аббревиатур? Вступление с проблемы? (проверяем структуру — финальные формулировки будут на шаге 7)
- [ ] Результаты мин. 8 сек? Лайфхак? Затравка? Промпты дословно?

### Выход
`content/<topic>/scenario.md` — план записи и ориентир, но НЕ финальный текст озвучки.

## Шаг 5. Режиссёр — ТЗ на запись

**Роль:** режиссёр записи экрана.
**Вход:** `scenario.md` из шага 4.

### Действия
1. Viewport: **1280x720** по умолчанию; другой размер — только если интерфейс остаётся читаемым. См. `docs/guides/recording.md`.
2. Блок «Подготовка»: закрыть лишние вкладки, скрыть персональные данные, расширения, VPN.
3. Для каждого шага: что открыть, куда кликнуть, что ввести (дословно), скролл, паузы, ожидаемый результат.
4. Параметры записи: viewport, формат, частота кадров.

### Самопроверка
- [ ] Точный текст для ввода? Паузы и скролл? Нет зависимости от попапов?
- [ ] Viewport указан? Подготовка с пунктом про персональные данные?

### Выход
`content/<topic>/recording-brief.md` — только инструкции для записи (действие + результат + текст для ввода), без озвучки и тайминга. В начале — блок «Подготовка» (viewport, закрыть лишнее, VPN).

### ПАУЗА
«Запиши видео по ТЗ: content/<topic>/recording-brief.md. Когда готово — скинь путь к файлу.»

## Шаг 6. Контролёр — анализ видео

**Роль:** контролёр качества. **Вход:** путь к записанному видео.

### Действия
1. Проверить наличие ffmpeg (`which ffmpeg`, если нет — `brew install ffmpeg`).
2. Проверить длительность через ffprobe: **максимум 10 минут** (см. `recording.md`). Если длиннее — вернуть на перезапись.
3. Извлечь кадры каждые 5 сек: `ffmpeg -i INPUT -vf "fps=0.2" frames/frame_%04d.png`
4. Сверить с scenario.md: все шаги, нет ошибок, генерация завершена, результат виден.
5. Вердикт: **OK** -> шаг 7 | **Частично** -> список на перезапись | **Плохо** -> агент (шаг 5) пишет новое recording-brief.md, пользователь переснимает.

Максимум 2 перезаписи. При проблемах — `issues.md` (попытка, проблемы, инструкции). После 2/2 — вернуться к шагу 4 или 5.

## Шаг 7. Монтажёр — анализ видео, озвучка, сборка

**Роль:** монтажёр (ffmpeg, edit.py, ElevenLabs TTS).
**Принцип:** видео первично, озвучка вторична. AI смотрит кадры реальной записи
и пишет текст под то, что видит. `scenario.md` — ориентир, не жёсткая рамка.

### Фаза A: Анализ видео и написание озвучки

1. Прочитать `docs/guides/voiceover-writing.md`.
2. Извлечь кадры каждые 5 сек: `ffmpeg -i INPUT -vf "fps=0.2" frames/frame_%04d.png`
   Прочитать кадры через Read tool: сначала ключевые (1 на шаг из scenario.md), затем детально проблемные участки. При >50 кадров — батчами по 10–15, сохраняя промежуточные выводы в файл.
3. Построить таймлайн: действия, переходы, проблемы (вырезать), результаты (замедлить).
4. Написать `voiceover-script.md` — для каждого сегмента: name, start, end, speed, текст.

Формат `voiceover-script.md`:

| # | Имя | Raw start | Raw end | Описание видео | Текст озвучки |
|---|-----|-----------|---------|----------------|---------------|
| 1 | 01_intro | 0.0 | 5.0 | Стартовый экран | Текст вступления... |
| 2 | 02_action | 5.0 | 22.0 | Клик, ввод текста | Описание действия... |
| 3 | 03_gen | 22.0 | 50.0 | ИИ генерирует (speed 6x) | Короткая фраза про генерацию... |

Speed указывается в описании видео для генераций; точные значения переносятся в edit.py.

### Скорости (approach v4)

Видео играет на натуральной скорости. Единственное исключение — генерация (ИИ печатает/рисует).
- **1.0x** — действия, результаты, лимиты, скролл, настройки (всё кроме генерации)
- **4–8x** — генерация (короткий комментарий поверх). Если после ускорения сегмент < 3 сек — бери меньший множитель.
- **Вырезать** — Finder, idle, контекстное меню, случайные клики
- **Озвучка под видео** — текст подгоняется под длительность сегмента, не наоборот. Если озвучка короче видео — тишина после слов (нормально). Без целевого хронометража — сколько получится.

### Контроль структуры

Сверить с `scenario-structure.md`: вступление, шаги, лайфхак, затравка (~15-20 сек), заключение. Затравка — исключение: пишется по сценарию, видео = заморозка последнего кадра.

### Фаза B: Сборка видео

1. Прочитать `docs/guides/video-branding.md`.
2. Сгенерировать `edit.py` с массивом SEGMENTS (из voiceover-script.md).

Формат SEGMENTS — список dataclass'ов:
- `VideoSegment(name, start, end, speed, text)` — нарезка из записи. `text` — озвучка (None = без голоса). `audio_file` — готовый аудиофайл вместо TTS.
- `FreezeSegment(name, text)` — озвучка поверх замороженного последнего кадра (затравка, заключение).

speed=1.0 натуральная, speed=4–8 генерация (сегмент после ускорения ≥ 3 сек). `edit.py` импортирует `EditConfig`, `VideoSegment`, `FreezeSegment`, `build` из `pipeline.pipeline.editor`.

3. Запустить edit.py: scale+speed -> сегменты, ElevenLabs TTS -> аудио, синхронизация, брендинг (интро 2с + аутро 2с + вотермарк), concat -> MP4.
4. Проверить: длительность 4:30-5:30, синхронизация, брендинг.

### Технические требования

**TTS:** ElevenLabs, модель `eleven_multilingual_v2`, голос из `ELEVENLABS_VOICE_ID` или `EditConfig.tts_voice_id`. Yandex SpeechKit — запасной (`tts_engine="yandex"`), Edge TTS — для черновиков. Пути: venv `pipeline/.venv/`, env в корне проекта.
**Аудио:** aac 48kHz mono. Брендинг-клипы с `anullsrc=r=48000:cl=mono`.
**Mux:** видео длиннее озвучки → `apad` + `-shortest`. Аудио длиннее → заморозить последний кадр (`-sseof -0.1` → loop → concat).
**Scale:** не кропить. `scale=1920:1080:force_original_aspect_ratio=decrease,pad=1920:1080:(ow-iw)/2:(oh-ih)/2:black`.

### Выход
`content/<topic>/final.mp4`

## Шаг 8. Контролёр — проверка монтажа

**Роль:** контролёр качества смонтированного видео.
**Вход:** `final.mp4` из шага 7, `voiceover-script.md`, `edit.py`.

### Действия
1. Извлечь кадры из собранного видео: `ffmpeg -i final.mp4 -vf "fps=0.2" work/review/frame_%04d.png`
2. Прочитать кадры через Read tool (ключевые → проблемные участки, батчами по 10–15). Сверить с `voiceover-script.md`: озвучка совпадает с экраном, нет чёрных кадров/артефактов, генерация ускорена.
3. ffprobe: длительность 4:30–5:30, аудио 48kHz mono, нет неожиданной тишины.
4. Брендинг: интро (~2 сек), аутро (~2 сек), вотермарк.

### Вердикт
**OK** → шаг 9. **Проблемы** → правки в `edit.py`/`voiceover-script.md`, пересборка, повтор. Макс. 3 итерации.

### Выход
`content/<topic>/final.mp4` (проверенный), `work/review/` (кадры).

## Шаг 9. Редактор — статья из видео

**Роль:** автор обучающих статей.
**Вход:** `final.mp4` (шаг 8) + `voiceover-script.md` + `edit.py`

### Действия
1. Прочитать `docs/guides/video-to-article.md` и `docs/guides/article-writing-style.md`.
2. Таймкоды в final.mp4: для каждого сегмента `dur = (raw_end - raw_start) / speed`, позиция = сумма dur всех предыдущих + ~2 сек интро. `ffmpeg -ss <позиция> -i final.mp4 -frames:v 1` → скриншот (1 на шаг).
3. Статья: озвучка → инструкция, промпты как код-блоки.
4. metadata.yaml: title (<60), description (<160), keywords (5-10), date, tool, level. Уровни: 1=ИИ-чаты, 2=Cursor+Claude Code, 3=Open Claw (concept.md).

### Самопроверка
- [ ] Нет маркеров ИИ-текста? Императив? Промпты как код-блоки? Alt-тексты? Самодостаточна без видео?

### Выход
`content/<topic>/article.md`, `article-images/step-*.png`, `metadata.yaml`
