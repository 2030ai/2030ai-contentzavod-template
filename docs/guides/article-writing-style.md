# Гайд: стиль текста для статей

Как писать тексты статей (`article.md`), чтобы они ранжировались в поисковиках и цитировались ИИ-чатботами.

## Контекст

Google и Яндекс не штрафуют за ИИ-происхождение текста. Штрафуют за низкое качество и массовость. Наша задача — писать ИИ-черновик, а потом доводить до уровня, неотличимого от экспертного текста.

Наше конкурентное преимущество — скриншоты из реальных интерфейсов. Это Experience из E-E-A-T, который ИИ не может подделать.

## Убрать маркеры ИИ-текста

### Слова и фразы, которых не должно быть

| Не писать | Почему | Замена |
|-----------|--------|--------|
| Важно отметить | Типичный маркер ChatGPT | Убрать или переформулировать |
| Стоит отметить | То же | То же |
| Давайте рассмотрим | То же | Начать сразу по делу |
| Безусловно / Конечно | Избыточная уверенность | Убрать |
| В данной статье мы | Канцеляризм | Убрать |
| Обеспечение / Привлечение / Использование | Отглагольные существительные | Обеспечить / Привлечь / Использовать |
| Является | Канцеляризм | Это / — |
| В рамках | Канцеляризм | В / при |
| Осуществлять | Канцеляризм | Делать |

### Структурные маркеры

| Признак ИИ | Что делать |
|-----------|------------|
| Все предложения одной длины | Чередовать: короткое (5 слов). Потом длинное с деталями и примерами. |
| Безупречная грамматика | Не нужны опечатки, но допустимы разговорные обороты |
| Ровный нейтральный тон | Добавить личную позицию: «мы рекомендуем», «по нашему опыту» |
| Линейная предсказуемая структура | Иногда начать с результата, потом объяснить |
| Нет сомнений — всё утверждается | Допустить «не всегда работает», «зависит от ситуации» |

## Что добавлять

### Реальный опыт (E-E-A-T)

- **Скриншоты** — наш главный актив. Каждый шаг = скриншот из реального интерфейса.
- **Конкретные промпты** — копируемые блоки с текстом, который мы вводили.
- **Практические наблюдения** — «обратите внимание: ChatGPT запомнил контекст из предыдущего чата» — это не ИИ-генерация, это наблюдение автора.
- **Оговорки и ограничения** — «если используете бесплатную версию, проекты могут быть недоступны» — ИИ этого не напишет.

### Авторство

- Указывать автора с биографией на странице.
- Не публиковать анонимно.

## Оптимизация для поисковиков и ИИ-чатботов

### Структура (влияет на цитирование)

- **Иерархия H2→H3→списки** — повышает шанс цитирования ИИ-чатботами на 40%.
- **Одна идея на абзац** — так ИИ-системы легче извлекают информацию.
- **Резюме/итого** в конце статьи — увеличивает шанс попасть в featured snippet.

### Определения

- Если в статье упоминается термин (проекты, промпт, инструкции) — дать краткое определение при первом упоминании.
- Контент с определениями цитируется ИИ-чатботами на 32% чаще.

### Разметка (при публикации на сайт)

- Schema: `Article`, `HowTo` — увеличивает цитирования на 28%.
- Дата публикации и дата обновления — контент, обновлённый за 30 дней, получает в 3.2 раза больше цитирований.

## Чего не делать

- Не публиковать raw output от ИИ без редактуры.
- Не штамповать 100+ статей в месяц (scaled content abuse).
- Не делать все статьи по одному шаблону с одним тоном.
- Не публиковать без фактчекинга — одна ошибка может повлиять на весь домен.

## Маркер «обновлено»

Каждая статья должна содержать два маркера даты:

```markdown
<!-- updated: 2026-02-28 -->

# Заголовок статьи

*Обновлено: 28 февраля 2026*
```

- `<!-- updated: -->` — машиночитаемый, для парсинга и автоматизации
- `*Обновлено:*` — видимый для читателя и поисковика

При любом изменении статьи — обновить обе даты. Реестр всех статей с датами: `pipeline/content-registry.md`.

## Чек-лист перед публикацией

- [ ] Нет слов-маркеров из таблицы выше
- [ ] Длина предложений разнообразна
- [ ] Есть скриншоты на каждый шаг
- [ ] Есть хотя бы одно личное наблюдение или оговорка
- [ ] Термины определены при первом упоминании
- [ ] Есть блок «Итого» / резюме
- [ ] Указан автор

## Примечание

SEO/GEO-правила меняются. Перед массовой публикацией проверьте актуальные требования поисковиков и площадки, на которой будет размещаться материал.
