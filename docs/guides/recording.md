# Запись экрана

Человек записывает экран вручную по `recording-brief.md`. Агент готовит сценарий, ТЗ, монтаж, озвучку, статью и проверки.

## Рекомендуемые параметры

- Viewport: 1280x720 или 1440x900, если интерфейс читаем.
- FPS: 30.
- Формат: `.mov` или `.mp4`.
- Длительность raw-видео: до 10 минут.
- Перед записью: скрыть персональные данные, токены, клиентские названия, приватные вкладки и расширения.

Для маленьких ноутбуков обычно безопасен viewport 1280x720. Не записывать 1920x1080, если элементы становятся слишком мелкими.

## Инструменты записи

- macOS встроенная запись экрана.
- OBS.
- FFmpeg, если пользователь умеет:

```bash
ffmpeg -f avfoundation -framerate 30 -i "1:none" \
  -c:v libx264 -preset ultrafast -crf 18 -pix_fmt yuv420p raw-recording.mp4
```

## Почему запись ручная

Браузерные и desktop automation tools плохо переносятся между интерфейсами, попапами, нативными диалогами ОС и IDE. Для публичного шаблона надёжнее: агент пишет точное ТЗ, человек записывает, агент монтирует.

## Извлечение кадров

После записи агент анализирует видео по кадрам:

```bash
cd pipeline
python -m pipeline.frames ../content/<slug>/raw-recording.mov -n 10
python -m pipeline.frames ../content/<slug>/raw-recording.mov -t 0:15 1:30 2:45
```

Кадры сохраняются рядом с видео или в папку из `-o`. Модуль автоматически уменьшает изображения до безопасного размера.
