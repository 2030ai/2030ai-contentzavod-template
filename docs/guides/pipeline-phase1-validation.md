# Пайплайн: Фаза 1 — Валидация идеи

Шаги 1–3 (режим А) или 2Б–3Б (режим Б). Полный обзор: `content-pipeline.md`.

---

## Шаг 1. Исследователь — генерация идей (только режим А)

### Роль
Ты — эксперт по аудитории ИИ-обучения. Понимаешь боли всех, кто работает
за компьютером: маркетологи, HR, менеджеры, аналитики, бухгалтеры, юристы,
секретари, руководители, предприниматели, фрилансеры, преподаватели (полный список: `docs/concept.md`). Знаешь какие задачи они решают и где ИИ реально
экономит время.

### Вход
Запрос пользователя: тема + инструмент (опционально аудитория).

### Действия
1. Прочитать `pipeline/content-registry.md` (опубликованные) и `content/README.md` (все статусы) — проверить нет ли дублей.
2. Прочитать `docs/concept.md` — освежить понимание аудитории, уровней
   и критериев выбора задач (секция «Принцип двухчастных мастер-классов»).
3. Если есть локальный банк `content/ideas.md` — проверить нет ли готовых идей.
4. Сгенерировать **10 идей**. Каждая идея:
   - Заголовок (как для YouTube — цепляющий, конкретный)
   - Задача: что конкретно делаем (одно предложение)
   - Аудитория: кому полезно
   - Боль: что плохо без этого (одно предложение)
   - Результат: что получит зритель за 5 минут
   - Шаги (кратко): перечислить 5–12 действий, которые зритель выполнит на экране.
     Если не набирается 5 шагов — идея слишком простая, не брать.
   - Питч: заголовок + задача + аудитория + результат (4 поля, по одному предложению)
5. Оценить каждую идею по 5 критериям из concept.md (массовость, регулярность,
   боль, контраст, двухуровневость) — по шкале 1-10 каждый.
6. Выбрать **топ-5** с наибольшим средним баллом.

### Самопроверка (до 3 итераций)
Для каждой из 5 идей проверить:
- [ ] Решает конкретную рабочую задачу (не абстрактный «обзор инструмента»)?
- [ ] Результат можно показать за 5 минут на экране?
- [ ] Зритель получит то, что сразу применит в работе?
- [ ] Набирается 5–12 смысловых шагов для сценария (основные действия пользователя, без вступления/лайфхака/затравки/заключения)?
- [ ] Нет дубля в `pipeline/content-registry.md` и `content/README.md`?
Если хоть один пункт — нет, заменить идею следующей из списка 10.

### Выход
Сохранить `content/<topic>/ideas.md` — все 10 идей с оценками + отметка
топ-5 для фокус-группы.

## Шаг 2 / 2Б. Фокус-группа — оценка идей

### Роль
8 субагентов, каждый имитирует конкретного представителя целевой аудитории.
Подробные описания персон: `docs/guides/focus-group.md`.

### Вход
- **Режим А (шаг 2):** топ-5 идей из `ideas.md` (шаг 1) — для каждой передаётся питч.
- **Режим Б (шаг 2Б):** одна идея пользователя, оформленная как питч (заголовок, задача, аудитория, результат).

### Действия
1. Прочитать `docs/guides/focus-group.md` — персоны и вопросы.
2. Запустить **8 субагентов параллельно** (через Agent tool). Каждый получает:
   - Свою персону (имя, профессия, возраст, контекст, боли)
   - Питчи (5 в режиме А, 1 в режиме Б)
   - 4 вопроса для оценки каждого питча:
     1. Релевантность для моей работы (1-10)
     2. Кликнул бы я на такой заголовок? (да/нет + почему)
     3. Что именно зацепило / не зацепило?
     4. Чего не хватает, что бы я хотел увидеть?

   Запустить **8 параллельных вызовов Agent tool** в одном сообщении:
   - Каждый вызов: `description="Focus group: <имя>"`, `subagent_type="general-purpose"`, `run_in_background=true`
   - В `prompt` подставить: данные персоны из focus-group.md + питчи + 4 вопроса
   Дождаться завершения всех 8 агентов, собрать ответы.

3. Собрать все 8 ответов.

### Выход
Сырые ответы 8 персон — сохранить в `content/<topic>/focus-group-raw.md`, передать контролёру на шаг 3/3Б.

Формат `focus-group-raw.md`:
```markdown
# Сырые ответы фокус-группы

## Марина (маркетолог, 32)

### Идея 1: <заголовок>
- Релевантность: 8/10
- Кликнул бы: да — <причина>
- Зацепило: ...
- Не хватает: ...

(повторить для каждой идеи)

---
## Алексей (руководитель, 41)
...
```

## Шаг 3. Контролёр — анализ фидбека фокус-группы (режим А)

### Роль
Ты — аналитик. Обрабатываешь результаты фокус-группы, ранжируешь идеи,
формируешь рекомендации.

### Вход
Ответы 8 персон из шага 2 + топ-5 идей из шага 1.

### Действия
1. Для каждой из 5 идей:
   - Посчитать средний балл релевантности (из 8 оценок)
   - Посчитать сколько персон кликнули бы (из 8)
   - Собрать сильные стороны (что зацепило)
   - Собрать слабые стороны (что не зацепило)
   - Собрать предложения по улучшению
2. Составить рейтинг 5 идей по совокупной оценке.
3. Для каждой идеи вынести вердикт:
   - **В работу** — средний балл > 7 ИЛИ кликнули бы 6+ из 8 (достаточно одного сильного сигнала)
   - **Доработать** — всё остальное (средний балл 5-7, или один критерий хороший а другой нет)
   - **Заблокировать** — средний балл < 5, кликнули бы < 3 из 8
4. Для лучшей идеи — написать бриф (brief.md).

### Выход
Сохранить `content/<topic>/focus-group-report.md`:
  - Рейтинг 5 идей с баллами
  - Вердикт по каждой (в работу / доработать / заблокировать)
  - Детальный фидбек по каждой идее
  - Рекомендация: какую идею брать и что в ней скорректировать

Сохранить `content/<topic>/brief.md` для лучшей идеи:
  - Заголовок
  - Инструмент
  - Задача
  - Аудитория
  - Боль
  - Результат за 5 минут
  - Обоснование выбора
  - Учтённый фидбек фокус-группы

Пример заполненного поля "Учтённый фидбек": «Марина: добавить конкретные цифры в заголовок.
Дмитрий: показать работу с таблицами. Решение: включить шаг с Excel-данными.»

### ПАУЗА
Показать пользователю `focus-group-report.md` и спросить:
«Вот отчёт фокус-группы. Какую идею берём в работу? Или доработать/пересмотреть?»

Дождаться решения пользователя. После одобрения — перейти к шагу 4.

## Шаг 3Б. Контролёр — доработка идеи по фидбеку (режим Б)

### Роль
Ты — аналитик. Обрабатываешь фидбек фокус-группы по одной идее.

### Вход
Ответы 8 персон из шага 2Б + исходная идея пользователя.

### Действия
1. Посчитать средний балл релевантности (из 8 оценок).
2. Посчитать сколько персон кликнули бы (из 8).
3. Собрать: что зацепило, что не зацепило, предложения по улучшению.
4. Вынести вердикт:
   - **В работу** — средний балл > 7 ИЛИ кликнули бы 6+ из 8 (достаточно одного сильного сигнала)
   - **Доработать** — всё остальное (средний балл 5-7, или один критерий хороший а другой нет)
   - **Заблокировать** — средний балл < 5, кликнули бы < 3 из 8
5. Если «В работу» или «Доработать» — написать бриф с учётом фидбека.
6. Если «Заблокировать» — НЕ создавать brief.md. Сообщить пользователю:
   «Идея не прошла фокус-группу (средний балл X, кликнули бы Y из 8).
   Рекомендация: выбрать другую идею или существенно переработать эту.»

### Выход
Сохранить `content/<topic>/focus-group-report.md` (фидбек по одной идее).
Сохранить `content/<topic>/brief.md` (бриф с учётом доработок — только если вердикт не «Заблокировать»).

### ПАУЗА
Показать пользователю фидбек и бриф:
«Вот что сказала фокус-группа о твоей идее. Бриф готов — берём в работу?»

Дождаться решения пользователя. После одобрения — перейти к шагу 4.
