# Гайд: структура сценария видеоурока

Как строить сценарий (`scenario.md`) для обучающего видео.

**Важно:** scenario.md — это план записи и ориентир для озвучки, но НЕ финальный текст.
Финальная озвучка (`voiceover-script.md`) пишется на шаге 7 (Монтажёр) после просмотра
реального видео — см. `pipeline-phase2-production.md`, шаг 7.

## Целевые параметры

- **Длительность финального видео:** 4:30–5:30
- **Структура:** вступление → шаги → лайфхак → затравка «следующий уровень» → заключение
- **Количество смысловых шагов:** 5–12 (не считая вступления, затравки и заключения)
- **Количество сегментов в edit.py:** 15–30 (шаги + генерации + результаты + затравка)

Точная длительность определяется на этапе монтажа (шаг 7 — Монтажёр) после анализа
реального видео. Тайминги ниже — ориентиры для планирования записи.

## Структура сценария

### 1. Вступление (10–15 сек)

Две части:
- **Проблема/контекст** (5–7 сек): зачем это нужно, какую задачу решает. Не «сегодня посмотрим X», а «если вы тратите время на Y — вот как это решить».
- **Что получим** (5–7 сек): конкретный результат к концу видео. «За 5 минут мы создадим... и получим...»

Плохо: «Сегодня разберёмся с проектами в ChatGPT.»
Хорошо: «Каждый раз объяснять ChatGPT контекст — это потеря времени. Проекты решают эту проблему. За 5 минут мы настроим проект, получим контент-план и напишем первый пост.»

### 2. Основные шаги (3:00–3:30)

Каждый шаг — это цикл:

```
Действие → Генерация (ускоренная) → Результат (нормальная скорость)
```

**Действие** (1x скорость): пользователь что-то делает — нажимает, вводит текст, выбирает. Голос объясняет что и зачем.

**Генерация** (4–8x скорость): ИИ генерирует ответ. Голос описывает что происходит и чего ждать. Не молчать — зритель должен понимать, что это ускорение. Правило: если после ускорения сегмент короче 3 сек — бери меньший множитель.

**Результат** (1x скорость): показать и объяснить результат. **Не спешить.** Проскроллить, остановиться на ключевом месте, прокомментировать.

#### Правило: результат важнее действия

На показ результата тратить больше времени, чем на само действие. Зритель пришёл увидеть, что получится — не как нажимать кнопки.

| Что | Время |
|-----|-------|
| Нажать кнопку + ввести текст | 3–5 сек |
| Генерация (ускоренная) | 5–8 сек |
| Показать и объяснить результат | 8–12 сек |

#### Правило: объединять по смыслу, не по кликам

Один «шаг» для зрителя — это одна логическая задача, а не один клик. «Создать проект и назвать его» — один шаг, хоть там 5 кликов.

### 3. Лайфхак / «А что если» (20–30 сек)

Перед заключением — практический бонус. Что-то, чего зритель не ожидал:
- Неочевидная возможность инструмента
- Частая ошибка и как её избежать
- Комбинация с другим инструментом
- «А если нужно X — вот как»

Это отличает полезное видео от скучного «нажмите сюда, потом сюда».

### 4. Заключение (10–15 сек)

Две части:
- **Что получили** (5–7 сек): перечислить конкретные результаты (не абстрактно «мы многое узнали»)
- **Зачем это нужно** (5–7 сек): одна фраза, которая закрепляет ценность. Привязать к реальной задаче зрителя.

### 5. Затравка «Следующий уровень» (15–20 сек)

Финальная секция после заключения. Берём задачу, которую зритель только что решил вручную, и показываем, как её можно решить масштабнее с Cursor и Claude Code.

**Формула:** «Мы сделали X руками → а с Cursor и Claude Code можно Y → и ещё Z, о чём вы даже не думали».

**Правила:**
- Конкретно привязать к теме мастер-класса (не абстрактно «можно автоматизировать»)
- Показать расширение задачи, а не просто ускорение (не «быстрее», а «больше и круче»)
- Упомянуть MCP-интеграции, если применимо (публикация в соцсети, подключение к сервисам)
- Закончить затравкой на следующие видео

**Если тема уже про Cursor/Claude Code:** затравку делать про MCP-интеграции, продвинутые возможности или другой инструмент. Не повторять то, что уже показано в МК.

**Пример для контент-плана:**
> Мы сделали контент-план для одной кофейни за три минуты. А в Cursor с Claude Code можно описать задачу — и ИИ-агент сам напишет код, который сгенерирует план сразу на три площадки, с готовыми текстами всех постов, картинками и расписанием. А через MCP подключить нужные каналы публикации — и публиковать прямо из редактора. Как — покажу в следующих видео.

**Пример для резюме:**
> Мы адаптировали одно резюме под одну вакансию. А с Claude Code можно дать ему десять вакансий с hh — и он сам адаптирует резюме и напишет сопроводительные письма под каждую. А через MCP — подключиться к hh и отправить отклики прямо из редактора. Как — в следующих видео.

## Чек-лист перед записью

- [ ] Вступление называет проблему и обещает результат
- [ ] Каждый результат показан подробно (не менее 8 сек)
- [ ] Есть лайфхак или «а что если»
- [ ] Есть затравка «Следующий уровень» — конкретное расширение задачи через Cursor/Claude Code
- [ ] Затравка привязана к теме МК (не абстрактная), упоминает MCP если применимо
- [ ] Заключение перечисляет конкретные результаты
- [ ] Все промпты/тексты, которые вводятся — записаны дословно

Проверка озвучки по `voiceover-writing.md` делается на шаге 7, когда текст пишется
по реальному видео.
