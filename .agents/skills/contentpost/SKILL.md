# Генерация постов из идей в Telegram

## Конфигурация

Прочитай `.env` файл проекта для получения:
- `TELEGRAM_BOT_TOKEN` — токен бота
- `TELEGRAM_IDEAS_CHAT_ID` — ID чата с идеями
- `TELEGRAM_CONTENT_CHAT_ID` — ID чата контент-плана
- `OPENROUTER_API_KEY` — ключ OpenRouter для генерации изображений
- `OPENROUTER_IMAGE_MODEL` — модель для генерации изображений (по умолчанию `sourceful/riverflow-v2-pro`)

## Шаг 1. Забрать идеи из чата

Прочитай файл `state/last_update_id.txt` (если существует) — там хранится ID последнего обработанного обновления.

Запроси новые сообщения из чата идей через Telegram Bot API:

```bash
curl -s "https://api.telegram.org/bot{TOKEN}/getUpdates?offset={LAST_UPDATE_ID+1}&limit=100"
```

Из ответа отфильтруй только сообщения из чата идей (по `chat.id == TELEGRAM_IDEAS_CHAT_ID`).
Игнорируй сервисные сообщения (без поля `text`).

Если новых идей нет — сообщи пользователю и заверши работу.

## Шаг 2. Генерация и публикация

Прочитай `style/voice.md` один раз перед началом генерации.

Для каждой идеи из чата **сразу без паузы**:

### 2a. Исследование темы

Если идея — ссылка или краткая заметка, изучи тему через WebSearch / WebFetch, чтобы наполнить пост фактами и цифрами.

**Сохрани авторскую мысль:** если в сообщении с идеей есть текст от автора (не просто ссылка) — это уникальный авторский angle. Выдели его отдельно и передай субагентам как обязательный элемент. Пост должен строиться вокруг этой мысли, а не заменять её пересказом.

### 2b. Генерация текста (3 варианта через субагентов)

Запусти **3 субагента параллельно** через Task tool (`subagent_type: "general-purpose"`), каждому передай:
- Текст идеи и результаты исследования из шага 2a
- **Авторскую мысль из идеи** (если есть) с пометкой: «Эту мысль обязательно сохрани в посте как основу личного комментария или заголовка-тезиса. Не перефразируй до неузнаваемости.»
- Содержимое `style/voice.md` (скопируй в промпт)
- Инструкцию: сгенерировать пост строго по правилам скилла `authorstyle`

Каждый субагент должен сгенерировать свой вариант поста по структуре:
- 🖼 + заголовок-тезис
- Контекст
- Разбор через `—` (em-dash)
- **Личный комментарий** (обязательно!)
- Вывод
- Ссылка на источник
- Хештеги

**Важно:** в промпте каждому субагенту укажи: «Текст поста должен быть не длиннее 1024 символов, включая эмодзи, ссылку и хештеги.»

После получения 3 вариантов **выбери лучший** по критериям:
1. **Сохранена авторская мысль** из идеи (если была) — это приоритетный критерий
2. Соответствие стилю voice.md (тон, структура, приёмы)
3. Наличие конкретных фактов и цифр
4. Сильный личный комментарий (не шаблонный)
5. Укладывается в 1024 символа

### 2c. Получение изображения

Каждый пост **обязательно** публикуется с изображением. Приоритет поиска:

**Приоритет 1 — изображение из источника.**
Если в идее есть ссылка на статью/источник — открой страницу через Playwright (browser_navigate + browser_snapshot или browser_take_screenshot). Найди основное изображение статьи (hero image, og:image, первая крупная картинка). Скачай URL изображения.

**Приоритет 2 — изображение из других источников по теме.**
Если в источнике нет подходящего изображения — поищи через WebSearch изображения по теме. Предпочтение: официальные блоги, пресс-релизы, первоисточники. Открой найденные страницы через Playwright и найди подходящую картинку.

**Приоритет 3 — генерация через OpenRouter.**
Если подходящего изображения не найдено — сгенерируй через OpenRouter API (модель из `OPENROUTER_IMAGE_MODEL`):

```bash
curl -s -X POST "https://openrouter.ai/api/v1/chat/completions" \
  -H "Authorization: Bearer {OPENROUTER_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{OPENROUTER_IMAGE_MODEL}",
    "messages": [
      {
        "role": "user",
        "content": "<промпт для изображения>"
      }
    ]
  }'
```

Ответ содержит base64-изображение в content. Сохрани его в файл `output/images/{YYYY-MM-DD}_{slug}.png` (декодируй base64).

### Требования к изображениям для Telegram

- **Размер:** 1280×720 px (16:9) — оптимальный для предпросмотра в ленте Telegram
- **Формат:** JPEG или PNG, до 5 МБ
- **Стиль для генерации:** чистый, современный, технологичный. Без текста на изображении (текст будет в caption). Яркие акцентные цвета на тёмном или нейтральном фоне
- **Что работает в Telegram:**
  — Скриншоты интерфейсов и продуктов (повышают доверие)
  — Инфографика с 1-2 ключевыми цифрами
  — Фото реальных людей/событий из источника
  — Абстрактные tech-иллюстрации для концептуальных тем
- **Что НЕ работает:**
  — Стоковые фото с рукопожатиями и улыбающимися людьми
  — Перегруженные деталями изображения (мелкие элементы теряются на мобильном)
  — Изображения с мелким текстом — его не прочитают

### Промпт для генерации изображений

При генерации изображений составляй промпт на английском языке по формуле:

```
[Main subject], [style: modern/tech/minimalist], [composition: centered/wide],
[lighting: soft/dramatic], [color palette: dark background with accent color],
[mood: professional/futuristic], no text, no watermarks, 16:9 aspect ratio,
high quality, clean design
```

Примеры:
- Тема AI в образовании: `A futuristic classroom with holographic displays showing neural network diagrams, soft blue lighting, dark background, minimalist tech style, no text, 16:9, high quality`
- Тема стартапы: `Abstract visualization of a startup growth trajectory, glowing data points connected by light trails, dark navy background with orange accents, modern tech illustration, no text, 16:9`
- Тема датацентры: `Aerial view of a modern data center with blue LED lighting, rows of servers stretching into perspective, cinematic composition, dark atmosphere, no text, 16:9`

### 2d. Публикация с изображением

**Если есть URL изображения** (приоритет 1 или 2):

```bash
curl -s -X POST "https://api.telegram.org/bot{TOKEN}/sendPhoto" \
  -H "Content-Type: application/json" \
  -d '{
    "chat_id": "{TELEGRAM_CONTENT_CHAT_ID}",
    "photo": "<URL изображения>",
    "caption": "<текст поста>"
  }'
```

**Если изображение сгенерировано** (приоритет 3, файл на диске):

```bash
curl -s -X POST "https://api.telegram.org/bot{TOKEN}/sendPhoto" \
  -F "chat_id={TELEGRAM_CONTENT_CHAT_ID}" \
  -F "photo=@output/images/{filename}" \
  -F "caption=<текст поста>"
```

**Важно — один пост = одно сообщение:**
- Caption в Telegram ограничен 1024 символами
- Пост **обязан** уместиться в 1024 символа (это задаётся на этапе генерации в шаге 2b)
- Если выбранный вариант всё же превышает 1024 — **сократи текст** (убери наименее важные пункты из разбора, сожми формулировки), но НЕ отправляй два сообщения
- Каждая идея = ровно одно сообщение с фото + caption. Никогда не отправляй отдельное текстовое сообщение вслед за фото

Не используй `parse_mode` — отправляй plain text.

После успешной отправки:
1. **Удали обработанное сообщение из чата идей:**

```bash
curl -s -X POST "https://api.telegram.org/bot{TOKEN}/deleteMessage" \
  -H "Content-Type: application/json" \
  -d '{"chat_id": "{TELEGRAM_IDEAS_CHAT_ID}", "message_id": {MESSAGE_ID}}'
```

Если удаление не удалось (бот не админ или сообщение старше 48 часов) — не блокируй процесс, просто продолжай.

2. Сообщи пользователю кратко: какая идея → опубликован пост.

## Шаг 3. Сохранить состояние

После обработки всех идей сохрани ID последнего обработанного update в `state/last_update_id.txt`:

```bash
echo "{max_update_id}" > state/last_update_id.txt
```

## Шаг 4. Отчёт

Выведи итог:

```
Обработано: N идей
Опубликовано: N постов
Изображения: X из источников, Y сгенерировано
```

## Важно

- Не спрашивать одобрение — публиковать сразу
- Не показывать список идей — сразу генерировать и публиковать
- Каждый пост ОБЯЗАТЕЛЬНО с изображением — без картинки не публиковать
- Если идея — просто ссылка без контекста, исследовать по ссылке самостоятельно
- Не придумывать факты. Если данных мало — поискать в интернете
- При ошибке Telegram API — показать ошибку пользователю, не молчать
